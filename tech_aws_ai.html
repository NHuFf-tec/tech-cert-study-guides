Module 1: Cloud Concepts & AWS Global Infrastructure

Understanding what the cloud is, why organizations use it, and how AWS is physically organized worldwide.

CLF-C02 Domain 1: Cloud Concepts (24%)

What Is Cloud Computing?

AWS Definition of Cloud Computing

Cloud computing is the on-demand delivery of IT resources over the internet with pay-as-you-go pricing. Instead of buying and managing physical servers, you access compute, storage, databases, and hundreds of other services from AWS — paying only for what you use.

Cloud AdvantageWhat It MeansExample

Economies of ScaleAWS buys millions of servers; passes savings to customersStorage costs have dropped 80%+ over a decade
Stop Guessing CapacityScale up or down in minutes based on actual demandE-commerce site handles Black Friday spikes
Speed & AgilityNew IT resources available in minutes, not weeksDeveloper spins up a test server in 2 minutes
No Data Center ManagementAWS handles physical security, cooling, hardware maintenanceYour team focuses on applications, not infrastructure
Go Global in MinutesDeploy to multiple AWS regions worldwide instantlyLaunch in Tokyo same day as New York

Region Selection Criteria

  • Compliance & Data Sovereignty: EU GDPR requires EU data stays in EU → choose eu-west-1 or eu-central-1
  • Latency: Choose the region geographically closest to your primary users
  • Service Availability: Not all services launch in all regions simultaneously; some services (e.g., Bedrock) may only be in select regions
  • Pricing: Prices vary by region; us-east-1 is typically cheapest for many services

📝 Module 1 Quiz

1A startup wants to avoid buying physical servers and instead pay only for the compute they use each month. Which cloud advantage does this describe?
Trading CapEx for OpEx is the core economic shift of cloud computing. The startup avoids buying servers (capital expense) and instead pays a monthly bill (operational expense). This frees up capital and eliminates depreciation, maintenance contracts, and hardware refresh cycles. Exam tip: whenever the scenario describes "no upfront investment" or "pay monthly" → CapEx to OpEx.
2A healthcare company needs to keep patient records in their own data center for regulatory reasons, but wants to use AWS for analytics workloads. Which deployment model does this describe?
Hybrid cloud = on-premises infrastructure connected to public cloud. The patient records stay on-premises (compliance), while analytics runs in AWS. Connection is typically AWS Direct Connect (dedicated private fiber) or VPN over internet. Key signal words: "existing data center" + "also use AWS" = hybrid. Multi-cloud = using multiple cloud providers (AWS + Azure + GCP) — different concept.
3Which AWS infrastructure component is defined as "one or more data centers with redundant power and networking, identified as us-east-1a, us-east-1b, etc."?
Availability Zones (AZs) are the physical units within a Region. Each AZ has: independent power supply (different grid/UPS/generators), independent cooling, redundant networking, physical separation (miles apart but connected via low-latency fiber). Naming: Region code + letter suffix. Deploying across 2+ AZs = high availability. Regions contain AZs. Edge Locations serve CDN content. Local Zones extend specific services to new cities.

Module 2: Core AWS Services

The fundamental services you'll encounter in the exam and in every AWS environment: compute, storage, databases, and networking.

CLF-C02 Domain 3: Cloud Technology & Services (34%)

Compute Services

🖥️ Amazon EC2

Elastic Compute Cloud. Virtual servers in the cloud. You choose the instance type (CPU, RAM, storage). Like renting a computer by the hour. Full OS control — your responsibility to manage OS, patches, middleware.

⚡ AWS Lambda

Serverless compute. Run code without managing servers. Upload a function; AWS runs it when triggered. Pay only per invocation and duration (milliseconds). Auto-scales to zero — no idle costs.

🚢 Amazon ECS / EKS

Container services. ECS = Elastic Container Service (AWS-managed). EKS = Elastic Kubernetes Service (Kubernetes-managed). Run Docker containers at scale without managing the cluster infrastructure.

☁️ AWS Elastic Beanstalk

PaaS — platform as a service. Upload your application code; AWS handles the infrastructure (EC2, load balancer, auto-scaling, monitoring). Developer focus: just the code. Automatically provisions and manages resources.

⚖️ Elastic Load Balancer

Distributes incoming traffic across multiple EC2 instances across AZs. Application LB (HTTP/HTTPS), Network LB (TCP, ultra-high performance), Gateway LB (third-party virtual appliances).

📈 Auto Scaling

Automatically adjusts EC2 instance count based on demand. Define min/max/desired. Scale-out on high CPU; scale-in on low CPU. Works with ELB to maintain application performance and minimize cost.

EC2 vs Lambda: EC2 = you manage the server, pay by the hour even when idle, long-running apps. Lambda = no server management, pay per invocation, short-lived functions (<15 min). If the scenario says "run code without servers" = Lambda.

S3 Storage Classes — Cost Optimization

  • S3 Standard: Frequently accessed data; highest cost; 3 AZ replication
  • S3 Standard-IA (Infrequent Access): Monthly access; lower storage cost + retrieval fee
  • S3 One Zone-IA: Infrequent access; single AZ; lowest cost for IA; less resilient
  • S3 Intelligent-Tiering: Unknown access patterns; automatically moves objects between tiers based on usage
  • S3 Glacier Instant Retrieval: Archives accessed quarterly; millisecond retrieval
  • S3 Glacier Deep Archive: 7–10 year retention; 12-hour retrieval; cheapest storage on AWS

📝 Module 2 Quiz

1A developer needs to run a short function that processes image uploads whenever a new file lands in S3, without managing any servers. Which AWS service is the best fit?
Lambda is the answer when: short-lived code execution, triggered by events (S3 upload, API call, schedule), no server management needed. The S3 → Lambda trigger is one of the most common serverless patterns. Lambda function runs, processes the image, terminates — you pay only for execution time. EC2 would be overkill (always-on server). Beanstalk deploys full applications. ECS runs containers (longer-lived services).
2A social media app needs to store 100 billion user posts with single-digit millisecond read latency at any scale, using a flexible schema (not all posts have the same fields). Which AWS database service fits best?
DynamoDB is designed for exactly this: massive scale, flexible (schemaless) documents, consistent single-digit millisecond performance regardless of data size. RDS/Aurora = relational (fixed schema, SQL), better for structured data with relationships. Redshift = analytics/data warehouse, not real-time OLTP. DynamoDB runs Twitter's timelines, gaming leaderboards, and Amazon's own shopping cart — it's the go-to for extreme-scale applications.
3Which AWS storage service is described as "object storage with 11 nines of durability" and is used for storing images, videos, backups, and static website hosting?
Amazon S3 (Simple Storage Service) = the foundational object storage service. Key facts: 99.999999999% (eleven 9s) durability, unlimited total storage, objects up to 5TB, globally unique bucket names, multiple storage classes. EBS = block storage attached to EC2 (like a hard drive). EFS = managed file system mounted by multiple EC2s simultaneously. Storage Gateway = hybrid bridge between on-premises and S3/Glacier. S3 is one of the most-tested services on CLF-C02.

Module 3: AWS Security, Compliance & Pricing

Security is the #1 weighted domain on CLF-C02 (30%). Understanding the Shared Responsibility Model and IAM is essential.

CLF-C02 Domain 2: Security & Compliance (30%) CLF-C02 Domain 4: Billing & Pricing (12%)

The Shared Responsibility Model

🔑 The Most Important Concept on CLF-C02

AWS and customers share security responsibility, but for different things. "AWS is responsible for security OF the cloud; customers are responsible for security IN the cloud."

LayerAWS Responsibility ("of the cloud")Customer Responsibility ("in the cloud")
PhysicalData center buildings, physical servers, power, cooling, physical access controlNothing — you never touch AWS hardware
Hypervisor / Host OSVirtualization infrastructure, host operating systemNothing for managed services (Lambda, RDS); guest OS for EC2
Network InfrastructureGlobal fiber, routers, DDoS protection infrastructureVPC configuration, Security Groups, NACLs, firewalls you configure
Application / DataNothingYour application code, data encryption, user access controls, SSL certs, database contents
Identity & AccessIAM service itselfWho you grant IAM permissions to; rotating credentials; enforcing MFA
Managed Services (RDS, S3)Operating system, patching, underlying infrastructureData stored, access controls, encryption settings, network access
For EC2: Customer is responsible for the GUEST operating system (patching, firewall settings), applications on the instance, and data. AWS handles the physical host and hypervisor. For Lambda: AWS handles everything infrastructure-related; you're only responsible for your code and data.

AWS Identity & Access Management (IAM)

IAM ConceptWhat It IsBest Practice
Root AccountThe account created when you first sign up; has unlimited access to everythingNever use for daily work; enable MFA; create admin IAM user instead
IAM UserIndividual identity with long-term credentials (username/password + access keys)One user per person; no shared credentials; least privilege
IAM GroupCollection of IAM users; attach policies to group, not individual usersCreate groups by job function (Developers, Admins, Billing)
IAM RoleTemporary credentials assumed by services, applications, or users; no long-term passwordUse roles for EC2 instances, Lambda functions — never store access keys in code
IAM PolicyJSON document defining allowed/denied actions on specific resourcesUse AWS managed policies; apply least privilege; avoid wildcard (*) on sensitive resources
MFAMulti-Factor Authentication; second factor required to sign inEnable on root account and all privileged users; required for sensitive API calls

Principle of Least Privilege

Grant ONLY the permissions needed to perform a task — nothing more. A Lambda function that only reads from S3 should have an IAM role with ONLY s3:GetObject — not s3:* or *:*. This limits blast radius if credentials are compromised.

Key AWS Security Services

🛡️ AWS Shield

DDoS protection. Shield Standard: free, automatic for all AWS customers. Shield Advanced: paid, enhanced protection, 24/7 DDoS Response Team, financial protection against DDoS-related charges.

🔥 AWS WAF

Web Application Firewall. Filters HTTP traffic to protect against SQL injection, XSS, bot attacks, and other Layer 7 threats. Works with CloudFront, ALB, API Gateway.

🔑 AWS KMS

Key Management Service. Creates and controls encryption keys used to encrypt your data. Integrates with S3, EBS, RDS, and most AWS services. Customer Master Keys (CMKs) — you control, AWS manages infrastructure.

🔍 Amazon GuardDuty

Intelligent threat detection. ML-based; analyzes CloudTrail, VPC Flow Logs, DNS logs to detect anomalies like unusual API calls, crypto-mining, compromised instances. No agents needed.

📋 AWS CloudTrail

Audit log of all AWS API calls. Who did what, when, from where. Enabled by default. Logs: console actions, CLI commands, SDK calls, service-to-service calls. Essential for security audits and compliance.

🔎 AWS Config

Configuration compliance tracking. Records and evaluates AWS resource configurations over time. Did someone remove MFA? Did S3 bucket become public? Config detects and can auto-remediate.

🔐 AWS Secrets Manager

Stores and rotates secrets. Database passwords, API keys, OAuth tokens — stored encrypted, retrieved by apps at runtime. Eliminates hardcoded credentials. Auto-rotates RDS passwords.

🏢 AWS Organizations

Manages multiple AWS accounts. Consolidated billing (one invoice). Service Control Policies (SCPs) — guardrails that restrict what accounts can do. Separate accounts by environment (dev/staging/prod) or business unit.

📝 Module 3 Quiz

1Under the AWS Shared Responsibility Model, who is responsible for patching the operating system on an Amazon EC2 instance?
EC2 is IaaS — AWS manages the physical host, hypervisor, and networking infrastructure. The customer manages everything INSIDE the virtual machine: guest OS installation, OS security patches, application stack, firewall rules (Security Groups), and data. Contrast with RDS: AWS patches the database engine and OS because it's a managed service. EC2 = you manage the OS. RDS = AWS manages the OS and DB engine. This distinction appears frequently on CLF-C02.
2A company wants to save up to 72% on EC2 costs for a database server that will run 24/7 for the next 3 years. What pricing model should they use?
Reserved Instances provide up to 72% savings vs On-Demand for a 1 or 3-year commitment. This database runs 24/7 for 3 years = perfect steady-state workload for RIs. Spot Instances: up to 90% discount BUT can be terminated with 2-minute notice — NOT suitable for a production database. On-Demand: no discount. Dedicated Hosts: for licensing compliance. The 3-year all-upfront RI gives maximum discount for committed workloads.
3Which AWS service provides a complete audit trail of all API calls made in your AWS account — who did what, when, and from where?
CloudTrail = audit log of API calls. Every action in AWS (console click, CLI command, SDK call, cross-service call) is logged with: who (IAM user/role), what (action), when (timestamp), where (source IP). Essential for: security audits, compliance, incident investigation ("who deleted that S3 bucket?"). CloudWatch = performance metrics and logs. GuardDuty = threat detection using ML on CloudTrail/VPC logs. Config = tracks resource configuration changes over time. CloudTrail is enabled by default for 90 days of event history.

Module 4: AWS Well-Architected Framework

The six pillars that guide building secure, high-performing, resilient, and efficient cloud architectures. Every AWS architect lives by these.

CLF-C02 Domain 3: Cloud Technology (34%)

The Six Pillars

Mnemonic: OSRPCS — "Operators Should Rarely Produce Cheap Software"

Operational Excellence · Security · Reliability · Performance Efficiency · Cost Optimization · Sustainability

⚙️ Operational Excellence

Run and monitor systems to deliver business value. Perform operations as code. Make frequent, small, reversible changes. Anticipate failure. Learn from failures.

🔐 Security

Protect data, systems, and assets. Apply security at all layers. Enable traceability. Automate security best practices. Protect data in transit and at rest. Prepare for security events.

🏗️ Reliability

System ability to recover from failures and meet demand. Automatically recover from failure. Test recovery procedures. Scale horizontally. Stop guessing capacity. Manage change through automation.

⚡ Performance Efficiency

Use computing resources efficiently. Select the right resource types. Monitor performance. Make trade-off decisions. Democratize advanced technologies. Go global in minutes.

💰 Cost Optimization

Avoid unnecessary costs. Implement Cloud Financial Management. Adopt a consumption model. Measure overall efficiency. Analyze and attribute expenditure. Use managed and application-level services.

🌱 Sustainability

Minimize environmental impact. Understand your impact. Maximize utilization. Anticipate and adopt more efficient offerings. Use managed services. Reduce downstream impact of your workloads.

AWS Cloud Adoption Framework (CAF)

Six CAF Perspectives

The AWS CAF helps organizations develop an action plan for cloud adoption. Six perspectives organized into two groups:

  • Business Capabilities: Business (strategy, ROI), People (HR, change management), Governance (risk, compliance)
  • Technical Capabilities: Platform (architecture, services), Security (controls, compliance), Operations (monitoring, incident management)

Exam tip: CAF is about organizational transformation — it's not just a technical framework. People and Governance perspectives are as important as Platform and Security.

📝 Module 4 Quiz

1An e-commerce application experiences high traffic spikes during holiday sales and low traffic at other times. Which Well-Architected pillar principle does auto-scaling directly address?
Auto-scaling addresses Cost Optimization (pay only for capacity in use) AND Performance Efficiency (match resources to demand). The key principle: "adopt a consumption model" — pay for what you consume, scale down when demand drops. Over-provisioning for peak means paying for idle servers 11 months a year. Auto-scaling eliminates this waste. It also touches Reliability (handles unexpected traffic spikes). When in doubt, if the question emphasizes elasticity and avoiding over-provisioning, Cost Optimization is the primary answer.
2A company wants to move an existing on-premises application to AWS as quickly as possible with minimal code changes — just moving the virtual machines to EC2. Which migration strategy is this?
Rehost (Lift and Shift): move the application as-is to EC2 — no code changes. Fastest migration strategy; achieves cloud cost benefits (Reserved Instances, managed networking) without rework. Good first step when speed matters or when applications are complex and poorly documented. Replatform = minor adjustments (e.g., move MySQL to RDS — same MySQL, managed differently). Refactor = redesign for cloud-native (microservices, serverless) — maximum benefit but maximum effort. Repurchase = replace with SaaS.
3Which AWS tool automatically analyzes your actual EC2 usage and recommends right-sizing to reduce costs without impacting performance?
AWS Compute Optimizer uses ML to analyze actual utilization metrics (CPU, memory, network) and recommends the optimal instance type. "You're running m5.2xlarge but only using 10% CPU — downsize to m5.large and save 75%." Pricing Calculator: estimate costs before deployment. Budgets: set and alert on spending limits. Cost Explorer: analyze and visualize actual spending. Compute Optimizer: right-size specific resources. The exam distinguishes these tools by purpose.

Purpose-Built AI Services (No ML Expertise Needed)

Computer Vision. Detect objects, scenes, faces, text in images and video. Facial comparison, content moderation, celebrity recognition, PPE detection. Used by news agencies and security systems.

📝 Amazon Textract

Document intelligence. Extract text, forms, tables, signatures from scanned documents (PDFs, images). Goes beyond OCR — understands document structure. Healthcare forms, financial documents, legal contracts.

1A company wants to build a chatbot using Amazon Bedrock that answers questions about their internal HR policies. The chatbot must only answer based on approved HR documents — not general knowledge — to prevent inaccurate or hallucinated answers. Which Bedrock feature should they use?
Bedrock Knowledge Bases = managed RAG pipeline. You connect S3 (containing HR PDFs), Bedrock automatically chunks, embeds, and stores them in a vector database. When a question comes in: relevant document chunks are retrieved → sent to the FM as context → answer is grounded in your HR documents. This prevents hallucination and limits responses to approved content. Fine-tuning would teach the model HR terminology, but the model might still hallucinate; RAG grounds responses in actual documents. This is the most common enterprise GenAI deployment pattern.
2An AI model used for loan decisions is found to approve applications at a lower rate for people from certain zip codes that correlate with protected characteristics. Which Responsible AI risk does this represent, and which AWS tool helps detect it?
This is proxy discrimination (a form of bias): zip code is used as a proxy for race/ethnicity, producing discriminatory loan outcomes without explicitly using protected characteristics. SageMaker Clarify analyzes model predictions across demographic groups, measuring metrics like demographic parity, equalized odds, and disparate impact. It identifies which features contribute most to disparate outcomes. This is a major regulatory concern under US fair lending laws (ECOA, Fair Housing Act). The "technically not using race" defense fails when a proxy variable is used.
3Which prompt engineering technique involves providing 2–5 examples of the desired input-output format to help the model understand your task?
Few-shot prompting: provide examples. "Here are 3 examples of customer emails classified as urgent/not urgent: [example 1] → urgent. [example 2] → not urgent. [example 3] → urgent. Now classify: [new email]." The model learns the pattern from your examples. Zero-shot: no examples given; model must infer task entirely from instruction. Chain-of-thought: ask model to reason step by step. Role prompting: assign model a persona ("You are a senior software engineer..."). Few-shot is the most reliable technique for consistent structured outputs.

1What does "elasticity" mean in the context of AWS cloud computing?
Elasticity = dynamic scaling in both directions. Auto Scaling Groups add EC2 instances during peak traffic and remove them during low traffic. You pay only for running instances. This is distinct from scalability (ability to handle increased load) and high availability (resilience to failure). Lambda is inherently elastic — it scales to zero when not used. Elasticity is one of the key economic advantages of cloud vs on-premises (where you must provision for peak permanently).
2A company in Germany must ensure customer data never leaves the European Union due to GDPR compliance. Which AWS infrastructure consideration is most relevant?
Data sovereignty is a primary consideration for EU companies under GDPR. AWS data stays in the region you deploy to — eu-central-1 (Frankfurt) or eu-west-1 (Ireland) ensure data remains in the EU. AWS does not move data between regions without your permission. Additional compliance tools: AWS Artifact (compliance reports), AWS Config (track data storage configurations), Customer Managed Keys with KMS (control over encryption). GDPR compliance is a shared responsibility — AWS provides compliant infrastructure; customers configure it correctly.
3Under the Shared Responsibility Model, which of the following is ALWAYS the customer's responsibility regardless of whether they use EC2, RDS, or Lambda?
Data access control and encryption are ALWAYS customer responsibilities regardless of service. You decide who can access your S3 bucket, your RDS database, your Lambda's environment variables. AWS never grants access to your data; you configure IAM policies and encryption settings. Patching varies: EC2 guest OS = customer; RDS engine = AWS; Lambda runtime = AWS. Physical infrastructure = always AWS. The customer always owns the data and is responsible for access control to that data.
4Which IAM best practice prevents a single compromised credential from granting unlimited AWS account access?
Least privilege limits blast radius: if a Lambda function's IAM role is compromised but only has s3:GetObject on one bucket, the attacker can only read that bucket. Compare: if an admin user is compromised, the attacker owns your entire AWS account. Best practices: separate IAM roles for each service; no wildcard (*) permissions on sensitive resources; enable MFA on all human users; rotate access keys; use IAM roles (not access keys) for applications. Never share credentials — one identity per person/service.
5A company needs to connect their on-premises data center to AWS with consistent, dedicated bandwidth (not shared internet) for a financial application processing millions of transactions daily. Which service provides this?
AWS Direct Connect = dedicated private fiber connection between your data center and AWS (bypasses the public internet). Consistent bandwidth, lower latency, and higher security than VPN. Used for: financial services (consistent performance), large data transfers (avoid internet bandwidth costs), compliance (data never touches public internet). VPN = encrypted tunnel over the public internet (shared, variable latency). Direct Connect speeds: 50Mbps to 100Gbps. Often paired with VPN as backup. Higher cost than VPN but much more reliable.
6An e-commerce company with steady 24/7 web server traffic wants maximum cost savings on their EC2 instances. What pricing model should they use?
Steady-state 24/7 workload = Reserved Instances. The trade: commit to 1 or 3 years → save up to 72%. You know you'll always need these servers, so the commitment is safe. Spot Instances: up to 90% savings BUT can be terminated with 2-minute notice — NOT suitable for a 24/7 customer-facing website (would cause outages). On-Demand: no discount. Free Tier expires after 12 months and only covers small resources. The rule: predictable, continuous workloads → Reserved; unpredictable or fault-tolerant → Spot.
7Which AWS service monitors your AWS environment using ML to detect anomalies like unusual API calls, compromised EC2 instances performing crypto-mining, or exfiltration of S3 data?
GuardDuty = intelligent threat detection using ML. It analyzes CloudTrail logs, VPC Flow Logs, and DNS logs to find anomalies: API calls from unusual countries, unusual data transfer volumes, known malicious IPs, reconnaissance patterns. Enables threat detection with no agents, no configuration. CloudTrail = audit log of API calls (raw data). Config = configuration compliance (did settings change?). Amazon Inspector = vulnerability scanning (does my EC2 have unpatched CVEs?). GuardDuty connects the dots to find active threats.
8Which AWS cost tool should a company use to visualize their current AWS spending by service, identify which team is spending the most, and forecast next month's costs?

Know these four tools and when each is used.
9According to the AWS Well-Architected Framework, deploying your application across multiple Availability Zones primarily satisfies which pillar?
Multi-AZ deployments are the primary tool for Reliability. The Reliability pillar's core concern: the system recovers from failure and continues meeting demand. If AZ-a loses power, AZ-b and AZ-c continue serving traffic. RDS Multi-AZ: automatic failover to standby in another AZ within minutes. EC2 Auto Scaling across AZs: new instances launch in healthy AZs if one fails. ELB routes traffic only to healthy AZs. Performance Efficiency: right-sizing and choosing appropriate services. Cost Optimization: eliminating waste. Reliability: recovering from failures.
10A company plans to replace their outdated on-premises CRM system with Salesforce and move their email to Microsoft 365. What migration strategy is this?

You're not migrating the old application — you're replacing it entirely with a vendor-managed SaaS product. This eliminates infrastructure management but requires data migration and user retraining. Rehost = move existing app to EC2 unchanged. Replatform = minor improvements during move (e.g., self-managed MySQL → RDS). Refactor = full redesign for cloud-native architecture.
11What is the primary advantage of Amazon Aurora over Amazon RDS for MySQL in high-demand production environments?
Aurora = AWS's high-performance relational database. MySQL and PostgreSQL compatible (drop-in replacement) but reengineered on a distributed storage system. 5x MySQL performance, 3x PostgreSQL performance. Storage: 6 copies automatically maintained across 3 AZs; if one AZ fails, data is safe. Aurora Serverless: automatically scales compute up and down; pay per second; ideal for variable workloads. Higher price than RDS but significantly better performance and availability. RDS is simpler and cheaper for moderate workloads; Aurora for demanding production systems.
12Which AWS AI service would a news media company use to automatically generate closed captions for their video content in multiple languages?

13What is Amazon Bedrock's key differentiation from building directly with open-source models on EC2?
Bedrock's advantages over self-hosting: (1) No GPU management — Bedrock handles all underlying infrastructure; (2) Model choice — Claude, Titan, Llama, Stable Diffusion, Mistral, Cohere all accessible via ONE API; (3) Enterprise security — VPC endpoints, IAM, encryption, audit logs; (4) Data privacy — your prompts and data are NEVER used to train or improve the underlying FM providers' models; (5) Built-in features — Guardrails, Knowledge Bases, Agents. Trade-off: less flexibility and higher per-call cost than self-hosting, but massively simpler operationally.
14Which prompt engineering technique is most likely to improve an LLM's accuracy on a complex multi-step math or reasoning problem?
Chain-of-thought (CoT) prompting dramatically improves performance on complex reasoning tasks. Adding "think step by step" or "show your work" causes the model to generate intermediate reasoning steps before the final answer. Why it works: forces the model to allocate more "compute" to the problem by generating more tokens. Research shows CoT can improve accuracy on math/logic problems from 18% to 80%+. Most effective with larger models. Used in: financial analysis, legal reasoning, diagnostic support, complex coding. Simple classification tasks don't need CoT.
15What is "model drift" and how does Amazon SageMaker Model Monitor help?
Model drift (also called data drift or concept drift) is a critical MLOps concern. Example: a fraud detection model trained on 2020 transaction patterns becomes less accurate as attack patterns evolve by 2023. Model Monitor continuously compares incoming inference data to the training data baseline, calculating statistical drift metrics. When drift exceeds a threshold: alert sent, model retrained. Without monitoring, a deployed model silently degrades. As a manager, you need SLA agreements for model performance — just as you have SLAs for system uptime. Model lifecycle management is a key responsible AI practice.
16A company is evaluating whether to migrate from on-premises servers to AWS. Which tool helps them quantify the 5-year total cost comparison including server hardware, power, cooling, and IT staff?

Typically shows 20–60% savings when factoring all true on-premises costs (many companies underestimate). Pricing Calculator: estimate specific AWS service costs. 17Which AWS service enables you to set organizational guardrails that PREVENT member accounts in your AWS Organization from provisioning resources in unapproved regions or disabling CloudTrail logging?
Service Control Policies (SCPs) in AWS Organizations are maximum permission boundaries for entire accounts. Unlike IAM policies (which apply to users/roles within an account), SCPs apply to the entire AWS account and override even admin IAM policies. Example SCP: "Deny any action that would disable CloudTrail" or "Deny any action outside eu-west-1 and eu-central-1". Even the account root user cannot override an SCP. Use case: enterprise security team enforces compliance guardrails across 100s of accounts from the management account. IAM policies = within an account. SCPs = across accounts in an organization.
18Amazon Rekognition is used to automatically moderate user-uploaded images on a social platform. The system incorrectly flags images of dark-skinned individuals at higher rates. Which Responsible AI principle is violated and what is the root cause?
