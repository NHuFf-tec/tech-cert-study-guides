🌤️ Module 1: Cloud Concepts

The foundation of AZ-900 — what the cloud is, why it exists, and how it's organized.

What Is Cloud Computing?

Cloud computing is the delivery of computing services — servers, storage, databases, networking, software, analytics, and AI — over the internet ("the cloud"). Instead of owning hardware, you rent computing resources from a provider like Microsoft Azure, AWS, or Google Cloud, paying only for what you use.

The Five Characteristics of Cloud (NIST Definition)

1. On-demand self-service: Provision resources without human interaction from the provider.
2. Broad network access: Available from anywhere over the internet.
3. Resource pooling: Provider serves multiple customers from shared infrastructure.
4. Rapid elasticity: Scale up or down quickly, automatically.
5. Measured service: Pay only for what you use (metered).

🏅 AZ-900 Exam Tip — Shared Responsibility

The Shared Responsibility Model is heavily tested. Know what the customer is ALWAYS responsible for regardless of service type: data, user accounts, and access management. The provider is always responsible for physical infrastructure. The split in the middle (OS, networking, etc.) shifts depending on IaaS/PaaS/SaaS.

📝 Module 1 Quiz — Cloud Concepts

1A company uses Microsoft 365 for email and Office apps, Azure Virtual Machines for their web servers, and Azure SQL Database for their application. Which service model correctly describes Azure SQL Database?
Azure SQL Database is PaaS: Microsoft manages the underlying infrastructure, OS patching, SQL Server engine updates, and HA/DR. The customer only manages their databases, schemas, users, and data. Compare: Azure Virtual Machines (IaaS — you manage the OS), Microsoft 365 (SaaS — Microsoft manages everything). PaaS lets you focus on your application without worrying about server maintenance.
2A retail company expects a massive traffic spike every December for holiday sales. The rest of the year traffic is low. Which cloud benefit BEST addresses this scenario?
Elasticity is the ability to scale resources up and down dynamically based on demand. In traditional IT, the company would need to buy enough servers to handle the December peak — then those servers sit idle 11 months a year (wasted CapEx). With cloud elasticity, they scale up for December, scale down in January, and pay only for what they use. This is one of the most powerful and commonly cited cloud benefits.
3Under the Shared Responsibility Model for a SaaS application, which item is the customer ALWAYS responsible for?
Regardless of IaaS, PaaS, or SaaS, the customer is always responsible for: (1) their data, (2) user accounts and access permissions, (3) end user devices. In SaaS, Microsoft manages everything technical — but they cannot control who you give access to. If an admin gives a contractor excessive permissions and data is stolen, that's the customer's responsibility. Physical infrastructure is always the provider's responsibility.

📝 Module 2 Quiz — Azure Services

2A developer wants to host a web application in Azure without managing any operating system or servers. They just want to deploy their code. Which service is BEST?
Azure App Service is PaaS for web applications. You deploy code (ZIP, container, or Git push) and App Service handles OS, patching, scaling, load balancing, and certificates. Azure Virtual Machines (IaaS) requires you to manage the OS — more control but more work. App Service is the right answer when "no server management" is a stated requirement — a very common exam pattern.
2An organization wants a private, dedicated network connection from their corporate office to Azure that does NOT go over the public internet. Which service provides this?
Azure ExpressRoute provides a dedicated, private connection from your data center to Azure through a connectivity partner (like AT&T or Equinix). It does NOT traverse the public internet — providing lower latency, higher bandwidth, and stronger security guarantees than VPN. Azure VPN Gateway also connects on-prem to Azure, but uses encrypted VPN over the public internet. ExpressRoute is the premium option for high-security/high-bandwidth requirements.
3A company needs to store 50TB of unstructured files (images, documents, backups) in Azure cost-effectively. Which service is most appropriate?
Azure Blob Storage (Binary Large Object) is object storage built for unstructured data at massive scale — images, videos, documents, backups, log files. It has tiered pricing: Hot (frequently accessed), Cool (infrequent access, cheaper), and Archive (rare access, very cheap). For 50TB of mixed files, Blob Storage is the cost-effective default choice. Azure SQL is for structured relational data. Cosmos DB is for NoSQL application data. VMs are for compute workloads.

📝 Module 3 Quiz — Security & Governance

1A company wants to ensure no one can deploy Azure resources outside the European Union region. Which Azure feature enforces this?
Azure Policy is used to define and enforce organizational standards and compliance rules. A policy denying resource creation outside EU regions will automatically block any deployment attempt outside those regions — regardless of who tries. RBAC controls what actions users can perform, not WHERE. Cost Management handles billing. Conditional Access handles authentication conditions. Policy is the governance tool for infrastructure compliance.
2A company plans to run a predictable workload 24/7 for the next 3 years. Which Azure pricing option offers the best discount?
Reserved Instances (1 or 3-year) provide the best discount for predictable workloads. 3-year = up to 72% off pay-as-you-go pricing. 1-year = up to 40% off. The trade-off: you're committing — if requirements change, you're locked in (though you can sometimes modify or cancel with a fee). Spot pricing offers larger discounts (80%+) but can be evicted — only suitable for non-critical, interruptible workloads.
3An IT manager wants to understand the full cost comparison between their current on-premises data center and moving to Azure. Which Azure tool is designed for this?
The TCO Calculator is specifically designed for "should we move to cloud?" business case analysis. It models: on-premises hardware, power, cooling, facilities, IT staff, maintenance — vs. equivalent Azure services. The Pricing Calculator estimates the cost of specific Azure resources. Cost Management tracks actual current spending. Knowing which tool to use for which scenario is an AZ-900 exam staple.

The ML Workflow — How a Model is Built

  1. Define the problem — What question are we trying to answer? What does success look like?
  2. Collect and prepare data — Gather training data; clean, transform, and label it
  3. Choose and train a model — Select an algorithm; train on the prepared data
  4. Evaluate the model — Test on held-out data; measure accuracy, precision, recall
  5. Deploy the model — Make it available as an API or service for real-world use
  6. Monitor and retrain — Track performance over time; retrain when data shifts

Key ML Terms for AI-900

Features: Input variables used to make predictions (e.g., age, income, location).
Labels/Targets: The output you want to predict (e.g., "will this customer churn?" Yes/No).
Training data: Historical data used to teach the model.
Validation data: Held-out data to tune the model during training.
Test data: Completely unseen data used to measure final model performance.
Overfitting: Model performs well on training data but poorly on new data — memorized rather than learned.

📝 Module 4 Quiz — AI & ML Concepts

1A bank builds a model to detect fraudulent transactions by training it on historical transactions labeled "fraud" or "legitimate." What type of machine learning is this?
Supervised learning uses labeled training data (known inputs + known correct outputs). Here, each historical transaction has a label (fraud/not fraud). The model learns the pattern: which combinations of features (amount, location, time, merchant type) correlate with fraud. New transactions are then classified using this learned pattern. Classification (fraud/not fraud) and regression (predict a number) are the two main supervised learning tasks.
2What is the main risk of a model that performs extremely well on training data but poorly when deployed with real-world data?
Overfitting occurs when a model learns the training data too precisely — including noise and outliers — and fails to generalize to new data. Like a student who memorized the answers to practice tests but can't answer new exam questions. Prevention: use more training data, reduce model complexity, use regularization techniques, or apply dropout (in neural networks). Always evaluate on held-out test data that was never used in training.
3Which AI category would include a model that generates new text responses based on a user's question, like ChatGPT?
Generative AI creates new content. LLMs (Large Language Models) like GPT-4 and Azure OpenAI are trained on massive text datasets and learn to predict the most likely next word/token — effectively generating coherent text. Unlike traditional ML that classifies or predicts from existing categories, GenAI creates new outputs (text, images, code, audio). This is the technology behind ChatGPT, GitHub Copilot, DALL-E, and Azure OpenAI Service.

How GenAI works, Azure Copilot ecosystem, and applying responsible AI principles.

How Generative AI Works

LLMs — Under the Hood (Simply Explained)

A Large Language Model is trained on billions of documents from the internet and books. It learns statistical patterns: given the words "The sky is...", the most likely next word is "blue." At massive scale, this predicts not just single words but coherent paragraphs, reasoning steps, and even code.

Prompt Engineering: How you structure your input (prompt) to an LLM dramatically affects the output. Good prompts are: specific, include context, specify format, and use examples. As a manager, you'll use prompts to instruct AI tools — this is a real skill.

Tokens: LLMs process text as tokens (roughly 3/4 of a word). Costs and context windows are measured in tokens. GPT-4 has a 128,000-token context window — roughly 100,000 words in one conversation.

📝 Module 6 Quiz — GenAI & Responsible AI

1A manager uses Microsoft 365 Copilot to draft a performance review email based on notes. Which Responsible AI principle requires that the manager review and edit the AI-generated draft before sending it?
Accountability: humans must retain responsibility and oversight for AI outputs — especially in consequential decisions like performance reviews, hiring, or financial decisions. An AI can assist and draft, but a human must review, verify accuracy, and be responsible for the final content. This principle prevents "AI made me do it" as an excuse and ensures meaningful human control over AI-assisted processes.
2What is "prompt engineering" in the context of Generative AI?
Prompt engineering is the practice of designing effective inputs for AI language models. Effective techniques: be specific and clear, provide context, specify format ("respond as a bullet list"), use examples (few-shot prompting), set constraints ("in under 100 words"), assign a role ("you are an expert accountant"). As a technology manager, prompt engineering is a practical, immediately applicable skill — it determines how useful AI tools are in your workflow. No coding required.
3An AI chatbot is designed to answer customer questions on a retailer's website. A customer asks it a medical question about drug interactions. The chatbot answers confidently but incorrectly. Which two Responsible AI principles are most relevant?
This scenario violates: Reliability & Safety (the chatbot answered outside its domain incorrectly — potentially dangerous), and Transparency (users should know they're talking to an AI and understand its limitations). Best practice: AI systems should know their scope, decline out-of-scope queries gracefully, disclose AI identity, and redirect users to appropriate human experts for high-stakes decisions (medical, legal, financial). Hallucination — AI confidently stating false information — is the key reliability risk with LLMs.

1Which cloud deployment model allows resources to move between a company's private infrastructure and a public cloud provider?
Hybrid cloud combines public and private cloud environments, allowing workloads to move between them. Benefits: keep sensitive data on-premises, burst to public cloud for peak demand, modernize gradually. Azure tools: Azure Arc (manage on-prem resources from Azure), Azure ExpressRoute (private connectivity), Azure Stack (run Azure services on-prem).
2What does "high availability" mean in the context of Azure services?
High availability (HA) = maximizing uptime through redundancy. Azure achieves HA by running multiple instances across Availability Zones. Deploy across 2+ zones → 99.99% SLA. Single instance → 99.9% SLA. Azure Load Balancer and Traffic Manager route traffic away from failed instances automatically. HA is distinct from scalability (handling load) and disaster recovery (recovering from major outage).
3An Azure administrator needs to give a new employee read-only access to a specific Azure Storage account. Which feature should they use?
RBAC (Role-Based Access Control) assigns permissions to users, groups, or apps for specific Azure resources. Key roles: Owner (full control), Contributor (create/manage but no access assignment), Reader (view only). You can scope RBAC at management group, subscription, resource group, or individual resource level. Always use RBAC at the most granular scope needed — least privilege principle.
4Which Azure service continuously monitors your Azure environment for security vulnerabilities and misconfigurations?
Microsoft Defender for Cloud (formerly Azure Security Center) provides: Secure Score (overall security posture rating), vulnerability assessments, threat detection, regulatory compliance checks, and security recommendations. It scans your Azure resources and highlights misconfigurations (e.g., "this storage account is publicly accessible"). Azure Policy enforces rules but doesn't actively detect threats. Key Vault stores secrets securely. Azure Monitor tracks performance metrics.
5What is the Azure service that stores and manages sensitive information like API keys, passwords, and certificates?
Azure Key Vault is the secure secret management service. Never hardcode passwords, API keys, or certificates in application code — store them in Key Vault and reference them by name. Features: RBAC-controlled access, audit logging, automatic certificate renewal, HSM-backed encryption. This is a critical security practice: if credentials are in code, one leaked repository exposes all credentials. Key Vault keeps them separate and audited.
6Microsoft Entra ID (formerly Azure Active Directory) primarily provides what function?
Microsoft Entra ID is Microsoft's cloud identity platform. It manages: user accounts and passwords, group memberships, app registrations, MFA enforcement, Conditional Access policies, SSO (Single Sign-On) across thousands of apps. Every Microsoft 365 and Azure subscription uses Entra ID. It's different from on-premises Active Directory (which manages Windows domain authentication) — though they can be synced via Entra Connect.
7Which Azure pricing model offers the largest discount for a workload that runs continuously for 3 years?
For consistent, long-running workloads: Reserved Instances (1 or 3 years) give the best planned discount. 3-year = up to 72% off. 1-year = up to 40% off. Spot pricing gives larger discounts (up to 90%) but can be evicted anytime — only for fault-tolerant, interruptible workloads. Dev/Test pricing gives discounts on non-production environments. Pay-as-you-go is most flexible but most expensive per unit.
8What does Azure Availability Zones protect against?
Availability Zones protect against single data center failures. Each zone in a region is a physically separate building with independent power, cooling, and networking — connected by low-latency fiber. Deploying across 2+ zones means if one building has a power failure, your app continues running from the others. For entire region failure protection, use Azure Region Pairs (deploy to two different regions).
9A company trains an AI model using 10,000 historical sales records where each record has labeled outcomes (sale closed: yes/no). Which type of ML is this?
Supervised learning = labeled training data with known outcomes. Labeled sales records (features: customer size, industry, deal value + label: closed/not closed) train a classification model. This model then predicts likelihood of closing for new leads. Binary classification (yes/no, fraud/not fraud, churn/no churn) is the most common supervised learning task in business applications.
10Which Responsible AI principle is most relevant when an AI system makes decisions but the reasoning cannot be explained to affected parties?
Transparency means AI systems should be understandable and explainable. Explainability (XAI) is particularly critical in: credit decisions, hiring, medical diagnosis, legal proceedings — anywhere the decision significantly impacts a person. Regulations (EU AI Act, GDPR) increasingly require AI explainability. "Black box" AI that cannot explain its decisions creates legal and ethical risk. Azure provides tools like Responsible AI Dashboard for model explanation.
11Which Azure AI service would you use to automatically extract invoice number, total amount, vendor name, and line items from scanned PDF invoices?
Azure Document Intelligence (formerly Form Recognizer) is the purpose-built service for extracting structured data from documents. Pre-built models for: invoices, receipts, ID documents, tax forms, business cards. Custom models for unique document types. It goes beyond basic OCR (reading text) to understanding document structure and extracting named fields. Automates accounts payable, expense management, and any document-heavy business process.
12Azure Resource Groups serve what primary purpose?
Resource Groups are logical containers — like folders for your Azure resources. Best practices: group resources by application or project (all resources for "Project X" in one group), use the same region as the resources, and apply tags for cost tracking. You can deploy, manage, and delete all resources in a group at once. RBAC and Azure Policy can be applied at the resource group level to control access to everything within.
13What is a key advantage of serverless computing (like Azure Functions) compared to Azure Virtual Machines?
Serverless (Azure Functions, AWS Lambda) means: no server provisioning, no OS management, automatic scaling (including to zero), and pay per execution. If your function runs 10,000 times a month for 1 second each, you pay for ~2.8 hours of compute — not 720 hours (a full month VM). Perfect for: event-driven tasks, APIs with variable traffic, scheduled jobs. VMs are better for: persistent workloads, full OS control, stateful applications running continuously.
14A global retailer wants to serve product images to customers worldwide with minimum latency, regardless of where users are located. Which Azure service helps?
Azure CDN distributes content copies to hundreds of edge locations globally. A user in Singapore gets the image from a nearby CDN node rather than the origin server in the US — reducing latency from 200ms to 10ms. CDNs are essential for: static assets (images, CSS, JS), video streaming, software downloads. CDN also reduces origin server load by serving cached content. Azure CDN integrates directly with Azure Blob Storage and App Service.
15Which Azure tool should a business use BEFORE migrating to Azure to understand the total cost difference vs. current on-premises infrastructure?
TCO Calculator is for migration business cases — it compares on-premises costs (servers, storage, networking, power, cooling, facilities, IT staff, software licenses) vs. equivalent Azure costs. Azure Pricing Calculator: estimate cost of specific Azure services you're planning to use. Azure Cost Management: track and analyze actual current Azure spending. Azure Advisor: recommendations for cost optimization of existing resources. Know which tool is for which phase: before migration = TCO; designing = Pricing Calculator; managing = Cost Management.
16What is "overfitting" in machine learning?
Overfitting: high training accuracy, low test accuracy. The model "memorized" training data instead of learning patterns. Analogy: a student who memorizes only the practice exam answers but fails on different questions. Prevention: more diverse training data, simpler model architecture, regularization techniques, dropout, cross-validation. The opposite is underfitting (model too simple — performs poorly on both training and test data).
17Azure's Zero Trust security model is based on which core principle?
Zero Trust: the traditional "castle and moat" security model (trust everything inside the firewall) is obsolete in the cloud era. Zero Trust assumes breach: verify every identity, every time, from every location. Core pillars: verify explicitly (always authenticate/authorize), use least privilege (minimal necessary access), assume breach (monitor, detect, respond). Implemented via: MFA, Conditional Access, Privileged Identity Management, Microsoft Defender for Identity.
18Which of Microsoft's Responsible AI principles requires that AI-generated content or AI-assisted decisions be disclosed as such to users?
Transparency covers AI disclosure. If a chatbot is AI, users should know. If an AI made a decision affecting them (loan denial, content moderation), they should be able to understand why. The EU AI Act requires transparency for certain AI systems. Microsoft's guidelines: clearly label AI-generated content, provide explanations for AI decisions, make AI limitations known. This is increasingly regulated — as a tech manager, transparency in AI deployments protects your organization legally.
19Which Azure service enables you to build a custom chatbot that answers questions from your company's own documents and knowledge base?
RAG (Retrieval Augmented Generation) is the architecture: Azure Cognitive Search indexes your documents → when a user asks a question, relevant documents are retrieved → these are included in the prompt to Azure OpenAI → the LLM generates an answer grounded in YOUR documents. This prevents hallucination and limits answers to your approved content. Copilot Studio makes this accessible without coding. This is the most common enterprise AI deployment pattern in 2024–2025.
20What is the Azure service that provides the Secure Score — a measure of your overall Azure security posture?
Microsoft Defender for Cloud provides the Secure Score — a percentage rating of your Azure environment's security posture. Each recommended action has a score impact. Following recommendations (enable MFA, enable encryption, restrict public access) improves your score. It also provides regulatory compliance views (GDPR, PCI-DSS, ISO 27001 compliance status). As an IT manager, your Secure Score is a KPI for your team's security performance.
21An AI model determines a customer's creditworthiness. When audited, it is found to approve loans for identical applications at different rates based on the applicant's zip code (which correlates strongly with race). Which principle is most violated?
This is proxy discrimination — a classic AI fairness problem. The model wasn't explicitly given race, but zip code is a highly correlated proxy. This violates Fairness and potentially fair lending laws (ECOA, Fair Housing Act). Detection requires fairness audits: run the model on identical applications varying only the protected characteristic (or proxy). Azure Responsible AI Dashboard includes fairness assessment tools. This is why "AI is objective because it doesn't use race" is a myth — biased proxies produce biased outcomes.
22Azure Cognitive Search combined with Azure OpenAI is most commonly used to build what type of solution?
RAG architecture: Azure Cognitive Search indexes and retrieves relevant documents from your knowledge base → Azure OpenAI generates answers using those retrieved documents as context. This grounds the AI's responses in your actual content, preventing hallucination and allowing the AI to answer questions about your specific products, policies, or internal knowledge. This is the #1 most deployed enterprise AI pattern — used for internal help desks, customer service bots, and document Q&A systems.
23Which Azure service manages multiple subscriptions under a hierarchy for consistent policy and RBAC application across an entire organization?
Management Groups provide a governance hierarchy: Tenant Root Group → Management Groups → Subscriptions → Resource Groups → Resources. Policies and RBAC applied at a Management Group level automatically inherit down to all subscriptions and resources beneath it. Example: a "Production" management group with policy requiring all resources to be encrypted. Essential for large enterprises managing dozens of subscriptions. Azure Policy and RBAC are the tools; Management Groups provide the scope.
24What is the primary difference between Azure Monitor and Microsoft Defender for Cloud?
Azure Monitor: observability platform — collects metrics (CPU, memory, latency), logs, traces, and enables alerting on performance/availability conditions. Microsoft Defender for Cloud: security platform — assesses security posture, detects threats, provides Secure Score, evaluates regulatory compliance. As an IT manager: you use Azure Monitor to ensure your services are running well; Defender for Cloud to ensure they are secure. Both are essential — different concerns.
25A user asks an Azure OpenAI-powered chatbot "How do I make a bomb?" The chatbot declines and redirects to safety resources. Which combination of principles does this demonstrate?
Azure Content Safety (built into Azure OpenAI Service) detects and blocks harmful content categories: hate, violence, sexual, self-harm. This demonstrates: Reliability & Safety (the AI system behaves safely and refuses harmful requests) and Accountability (the organization using the API has configured safety filters and accepts responsibility for outputs). Microsoft's Azure OpenAI content filtering is on by default and required for all deployments — organizations must take accountability for appropriate use.
